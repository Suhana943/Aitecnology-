Model Evaluation: Accuracy Metrics

Published: July 9, 2026 | Category: ML

Once a machine learning model is trained, the next critical step is to determine how well it performs on unseen data. Accuracy metrics allow us to quantify this performance, moving beyond simple intuition to provide objective measures of reliability and effectiveness.

Understanding Evaluation Metrics

The choice of metric often depends on the specific problem, especially in classification tasks where the cost of different types of errors may vary.

  • Accuracy: The ratio of correct predictions to the total number of predictions. While intuitive, it can be misleading if classes are imbalanced.
  • Precision: Measures the accuracy of positive predictions. It answers: "Of all instances predicted as positive, how many were actually positive?"
  • Recall (Sensitivity): Measures the ability of the model to find all positive instances. It answers: "Of all actual positive instances, how many did the model correctly identify?"
  • F1-Score: The harmonic mean of precision and recall. It provides a single score that balances both concerns, making it ideal for imbalanced datasets.

Metrics in Practice

For regression models, we typically look at different metrics, such as Mean Squared Error (MSE) or R-squared, which measure how close predicted values are to actual numerical targets.

Quick Example

from sklearn.metrics import accuracy_score, precision_score, recall_score

# Assuming y_true are the actual labels and y_pred are model predictions
y_true = [0, 1, 1, 0, 1]
y_pred = [0, 1, 0, 0, 1]

# Calculate metrics
acc = accuracy_score(y_true, y_pred)
prec = precision_score(y_true, y_pred)
rec = recall_score(y_true, y_pred)

print(f"Accuracy: {acc:.2f}, Precision: {prec:.2f}, Recall: {rec:.2f}")
