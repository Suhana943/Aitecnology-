NumPy Broadcasting: Efficient Array Operations

Broadcasting is one of NumPy’s most powerful features. It allows you to perform arithmetic operations between arrays of different shapes without needing to manually replicate or reshape data. This mechanism is critical for performance in machine learning, where we often need to add a "bias" vector to a large matrix of weights.

Broadcasting describes how NumPy treats arrays with different shapes during arithmetic operations. It "stretches" the smaller array across the larger array so that they have compatible shapes for element-wise operations.

1. The Broadcasting Rules

For two arrays to be "broadcastable," their shapes must be compatible. NumPy compares the dimensions starting from the trailing (rightmost) dimension:

  • Alignment: If the arrays have a different number of dimensions, the shape of the one with fewer dimensions is padded with ones on its left side.
  • Compatibility: Two dimensions are compatible if they are equal, or if one of them is 1.
  • Expansion: If the dimensions are compatible, the smaller dimension is effectively "stretched" to match the larger one during the computation.

2. Practical Usage

import numpy as np

# Define a 2x2 matrix and a 1D vector
A = np.array([[1, 2], [3, 4]])
b = np.array([10, 20])

# Adds b to each row of A automatically via broadcasting
result = A + b

print("Result of A + b:")
print(result)
Why Use Broadcasting in AI?

In neural networks, you often have a matrix of inputs (N × M) and a bias vector of size (M). Instead of manually creating a matrix of the same size as your inputs to add the bias, NumPy broadcasts the vector across every row of the matrix automatically. This saves massive amounts of memory and CPU cycles.
