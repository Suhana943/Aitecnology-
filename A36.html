Model Evaluation: Precision and Recall

Published: July 9, 2026 | Category: ML

In classification problems, relying solely on "Accuracy" can be misleading, particularly when dealing with skewed or imbalanced datasets—such as detecting a rare disease or fraudulent transactions. Precision and Recall provide a more nuanced look at how a model handles specific classes.

Defining the Core Metrics

To understand these metrics, we first categorize predictions into a Confusion Matrix:

  • True Positive (TP): Correctly predicted positive.
  • False Positive (FP): Incorrectly predicted positive (Type I error).
  • False Negative (FN): Incorrectly predicted negative (Type II error).

Precision

Precision measures the exactness of the classifier. It answers: "Of all instances the model predicted as positive, how many were actually positive?"

Precision = TP / (TP + FP)

Recall (Sensitivity)

Recall measures the completeness of the classifier. It answers: "Of all actual positive instances in the data, how many did the model correctly identify?"

Recall = TP / (TP + FN)

The Trade-off

There is a fundamental trade-off between Precision and Recall. Increasing the threshold for classifying something as "positive" usually increases Precision but decreases Recall. Finding the right balance depends on your objective:

  • High Precision: Necessary when the cost of a False Positive is high (e.g., flagging an email as spam that is actually an important work document).
  • High Recall: Necessary when the cost of a False Negative is high (e.g., failing to detect a patient who actually has a disease).

Quick Example

from sklearn.metrics import precision_score, recall_score

y_true = [1, 0, 1, 1, 0, 1]
y_pred = [1, 0, 0, 1, 0, 1]

# Calculating the scores
precision = precision_score(y_true, y_pred)
recall = recall_score(y_true, y_pred)

print(f"Precision: {precision:.2f}")
print(f"Recall: {recall:.2f}")
