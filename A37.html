Feature Engineering: Scaling and Normalization

Published: July 9, 2026 | Category: Data

In machine learning, features often exist on vastly different scales (e.g., "Age" might range from 0 to 100, while "Annual Income" ranges from 20,000 to 200,000). Many algorithms—especially those reliant on distance metrics like K-Nearest Neighbors or optimization techniques like Gradient Descent—will be heavily biased toward features with larger absolute values. Scaling and normalization are essential preprocessing steps to ensure every feature contributes equally to the model.

Why We Scale Data

  • Algorithm Convergence: For models like Linear Regression and Neural Networks, gradient descent converges significantly faster when features are on a similar scale.
  • Distance Sensitivity: Algorithms like K-Nearest Neighbors and SVM calculate distances between data points. Without scaling, the feature with the largest magnitude dominates the distance calculation.
  • Interpretability: Scaling allows for easier comparison of feature importance in linear models.

Common Techniques

  • StandardScaler (Standardization): Transforms data to have a mean of 0 and a standard deviation of 1. It assumes your data follows a Gaussian (normal) distribution.
  • MinMaxScaler (Normalization): Rescales the data to a fixed range, typically [0, 1]. This is useful when you need bounded data and don't assume a normal distribution.

Quick Example

from sklearn.preprocessing import StandardScaler
import numpy as np

# Sample data
data = np.array([[100, 0.001], [200, 0.002], [300, 0.003]])

# Apply standardization
scaler = StandardScaler()
scaled_data = scaler.fit_transform(data)

print("Scaled Data:\n", scaled_data)
