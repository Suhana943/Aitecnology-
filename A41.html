Decision Tree Algorithms Explained

Published: July 9, 2026 | Category: ML

Decision trees are one of the most intuitive and powerful supervised learning algorithms. They mirror the human decision-making process by creating a flow-chart-like structure that splits data based on specific feature values until a final decision (or prediction) is reached.

How Decision Trees Work

A decision tree breaks down a dataset into smaller and smaller subsets while simultaneously building an associated decision tree. The process involves:

  • Root Node: The starting point that represents the entire dataset.
  • Internal Nodes: Points where the data is split based on a feature condition (e.g., "Is the age > 30?").
  • Leaf Nodes: The final output or decision (e.g., "Will buy" or "Will not buy").

The algorithm automatically chooses the best features to split on by calculating metrics like Gini Impurity or Entropy, which measure how "pure" the resulting nodes are after a split.

Advantages & Disadvantages

  • Pros: Extremely easy to interpret and visualize; requires little data preprocessing (no need for scaling).
  • Cons: Highly prone to overfitting, where the tree becomes too complex and captures noise in the training data rather than the underlying pattern.

Quick Example

from sklearn.tree import DecisionTreeClassifier
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.metrics import accuracy_score

# Load sample data
data = load_iris()
X_train, X_test, y_train, y_test = train_test_split(data.data, data.target, test_size=0.2)

# Initialize and train the model
clf = DecisionTreeClassifier(max_depth=3)
clf.fit(X_train, y_train)

# Predict and evaluate
y_pred = clf.predict(X_test)
print(f"Accuracy: {accuracy_score(y_test, y_pred):.2f}")
