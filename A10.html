Robust Python Exception Handling

By: Aitecnology Team | Published: July 9, 2026

पायथन प्रोग्रामिंग में जब आपका कोड किसी ऐसी परिस्थिति का सामना करता है जिसे वह संभाल नहीं पाता, तो वह एक एरर (Error) देता है। इस एरर को पायथन की भाषा में Exception (अपवाद) कहा जाता है। यदि हम इसे सही तरीके से संभालते नहीं हैं, तो हमारा प्रोग्राम तुरंत बंद (क्रैश) हो जाता है। इसी समस्या से बचने के लिए Robust Exception Handling (मजबूत अपवाद प्रबंधन) का उपयोग किया जाता है।

1. Try और Except का बुनियादी उपयोग

पायथन मेंExceptions को संभालने के लिए मुख्य रूप से try और except ब्लॉक का उपयोग किया जाता है। जो कोड एरर दे सकता है उसे हम try ब्लॉक में रखते हैं और एरर आने पर क्या करना है, वह except ब्लॉक में लिखा जाता है।

try:
    number = int(input("एक संख्या दर्ज करें: "))
    result = 10 / number
    print(f"परिणाम: {result}")
except ZeroDivisionError:
    print("त्रुटि: आप किसी भी संख्या को शून्य (0) से विभाजित नहीं कर सकते!")
except ValueError:
    print("त्रुटि: कृपया केवल एक वैध पूर्णांक संख्या ही दर्ज करें।")

2. Finally और Else ब्लॉक

मजबूत कोडिंग के लिए try-except के साथ दो और ब्लॉक महत्वपूर्ण होते हैं:

  • Else: यह ब्लॉक तब चलता है जब try ब्लॉक में कोई भी एरर नहीं आता है।
  • Finally: यह ब्लॉक हर परिस्थिति में चलता है, चाहे एरर आए या न आए। इसका उपयोग आमतौर पर रिसोर्स क्लीनअप जैसे फाइल बंद करने या डेटाबेस कनेक्शन को क्लोज करने के लिए किया जाता है।
try:
    file = open("data.txt", "r")
    content = file.read()
except FileNotFoundError:
    print("त्रुटि: निर्दिष्ट फ़ाइल नहीं मिली।")
else:
    print("फ़ाइल को सफलतापूर्वक पढ़ लिया गया है।")
finally:
    if 'file' in locals() and not file.closed:
        file.close()
        print("फ़ाइल को सुरक्षित रूप से बंद कर दिया गया है।")

3. कस्टम एक्सेप्शन बनाना (Custom Exceptions)

पायथन के इन-बिल्ट एरर्स के अलावा, आप अपनी व्यावसायिक आवश्यकताओं (Business Logic) के अनुसार खुद के कस्टम एरर भी बना सकते हैं। इसके लिए आपको Exception क्लास को इनहेरिट (Inherit) करना होता है।

class AgeRestrictionError(Exception):
    def __init__(self, age, message="इस सेवा के लिए आपकी आयु कम है!"):
        self.age = age
        self.message = message
        super().__init__(self.message)

try:
    user_age = int(input("अपनी आयु दर्ज करें: "))
    if user_age < 18:
        raise AgeRestrictionError(user_age)
except AgeRestrictionError as e:
    print(f"कस्टम त्रुटि: {e.message} (दर्ज की गई आयु: {e.age})")
Aitecnology Tip: हमेशा विशिष्ट एक्सेप्शन्स (Specific Exceptions) को ही हैंडल करें। जेनेरिक except Exception: का उपयोग करने से बचें, क्योंकि यह वास्तविक लॉजिकल बग्स (Bugs) को छुपा सकता है जिन्हें खोजना बाद में कठिन हो जाता है।
