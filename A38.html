Categorical Data Encoding Techniques

Published: July 9, 2026 | Category: Data

Most machine learning algorithms are mathematical in nature and require numerical input. However, real-world datasets are often filled with categorical data—such as "City names," "Product types," or "User feedback levels"—which exist as strings or labels. Encoding is the process of transforming these categories into numerical formats that algorithms can interpret.

Common Encoding Methods

Choosing the right technique depends on whether the categorical data has an inherent rank (ordinal) or not (nominal).

  • Label Encoding: Each unique category is assigned an integer value (e.g., "Apple" = 0, "Banana" = 1, "Cherry" = 2). Use for ordinal data where there is a clear rank. Avoid for nominal data.
  • One-Hot Encoding: Creates a new binary column for each unique category. Best for nominal data with no intrinsic order.
  • Ordinal Encoding: Similar to Label Encoding but allows you to manually specify the rank/order of the categories.

Quick Example

import pandas as pd

# Data with nominal categories
df = pd.DataFrame({'Color': ['Red', 'Blue', 'Green']})

# Apply One-Hot Encoding
encoded_df = pd.get_dummies(df, columns=['Color'])

print(encoded_df)
