Python Decorators for Code Enhancement

By: Aitecnology Team | Published: July 9, 2026

Decorators are one of Python's most elegant features. They allow you to modify or extend the behavior of functions or classes without changing their actual source code.

1. What are Decorators?

A decorator is a function that takes another function as an argument, adds some "extra" functionality to it, and returns a new version of that function. Think of it like a wrapper—it wraps your main code inside an extra layer.

2. Why are they important?

  • DRY Principle: You can write a feature (like logging) once and apply it to dozens of functions.
  • Clean Code: They keep your main business logic separate from "housekeeping" tasks.
  • Reusability: Easily attach or detach features using the @ symbol.

3. How to use them?

def my_decorator(func):
    def wrapper():
        print("Before the function runs")
        func()
        print("After the function runs")
    return wrapper

@my_decorator
def say_hello():
    print("Hello! This is the main function.")

say_hello()

4. Use in AI & ML

In AI/ML, decorators are used for:

  • Timing Training: Measuring how long a model takes to train.
  • Validation: Checking input data shapes before passing them to a neural network.
  • Caching: Using @lru_cache to speed up math-heavy preprocessing.

5. Benefits

  • Efficiency: Automates repetitive tasks.
  • Readability: Code stays focused on the logic.
  • Maintainability: Change logic in one place (the decorator) instead of everywhere.
Aitecnology Tip: When working with PyTorch or TensorFlow, use custom decorators to automatically handle device placement (CPU vs. GPU), making your training code much cleaner!
