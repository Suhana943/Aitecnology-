Handling Missing Data with Imputation

Published: July 9, 2026 | Category: Data

Missing data is an inevitable reality in real-world datasets, stemming from sensor errors, survey non-responses, or database integration issues. Simply dropping rows with missing values can lead to significant data loss and bias. Imputation—the process of replacing missing data with substituted values—allows us to retain valuable information and build more robust models.

Common Imputation Strategies

The strategy you choose depends on the nature of the data and the reason for the missingness:

  • Mean/Median Imputation: Replacing missing values with the mean (for normally distributed data) or median (for skewed data or data with outliers).
  • Mode Imputation: Replacing missing values with the most frequently occurring value. Used for categorical data.
  • Arbitrary Value Imputation: Replacing missing values with a placeholder, such as -1 or 999. Useful if "missingness" itself is a pattern.
  • K-Nearest Neighbors (KNN) Imputation: Using the values of the "nearest" complete data points to estimate the missing value based on feature similarity.

Why Impute?

Imputation prevents models from crashing and helps preserve the sample size, which is critical when dealing with small datasets.

Quick Example

from sklearn.impute import SimpleImputer
import numpy as np

# Data with a missing value
X = [[1, 2], [np.nan, 3], [7, 6]]

# Impute with the mean of each column
imputer = SimpleImputer(strategy='mean')
X_imputed = imputer.fit_transform(X)

print(X_imputed)
